Overview

Developed at DSTO Australia, Replay was originally created to visualize the Annex Object-Capability System - to help developers understand and debug the system as it was created. Since the Annex project is no longer active, Replay has been released as open source to allow others to use and contribute to it's ongoing development.

Replay provides a highly intuitive interface to allow the exploration of dynamic, interconnected systems.

Plugins

Replay includes a plugin system, to allow external software to extend the functionality of Replay, and to interface with external data sources.

API Documentation is generated for Replay when building from source using the --enable-gtk-doc option to configure. A copy of the API documentation for the current release of Replay is available here

Plugins written in both C and Python already ship with Replay which can be used as the basis for the development of further extensions.

Dependencies

Replay was designed primarily to run under the GNOME desktop for Linux and requires the following dependencies:

  • gnome-common
  • pkg-config
  • autoconf
  • automake
  • intltool
  • gettext
  • glib-2
  • gtk+-3
  • gtkglext-3
  • libpeas
  • yelp-tools (for building source code documentation with --enable-gtk-doc configure flag)

Packages

Packages for both Replay and GtkGLExt 3.0 on 64-bit Fedora are provided:

  • gtkglext-libs-2.99.0-1.fc18.x86_64.rpm
  • gtkglext-devel-2.99.0-1.fc18.x86_64.rpm (required to build Replay from source)
  • replay-0.8.0-1.fc18.x86_64.rpm
  • replay-devel-0.8.0-1.fc18.x86_64.rpm (required to build external plugins for Replay written in C)

Compiling from source

To build Replay from git:

git checkout https://github.com/alexmurray/replay.git
cd replay
./autogen.sh --prefix=/usr --enable-gtk-doc --enable-debug # for 64-bit Fedora add --libdir=/usr/lib64
make
sudo make install
